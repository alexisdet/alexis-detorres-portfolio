AdaptAI

AI-supported productivity workflows that turn scattered project communication into priorities, drafts, meeting context, and follow-up actions.

TeamHackathon Build
RoleProduct Strategy, UX/UI Design

Long Story Short

I designed AdaptAI to reduce project communication overload

AdaptAI helps professionals understand what needs a response, what requires follow-up, and what can become a status update. The product direction centered on Outlook, Teams, meeting preparation, email triage, and project status reporting.

01

Action Priority

Focused attention on messages that require decisions, responses, or follow-up instead of summarizing every thread.

02

Editable AI Drafts

Kept users in control with draft, reword, review, send, and delete options before communication goes out.

03

Meeting Context

Connected email, Teams, milestone, and meeting context so users can enter conversations prepared.

How might we help professionals move from scattered updates to clearer project actions?

Workflow Research

To reduce communication friction, I mapped where project context gets lost

The product direction was shaped around recurring workflow pain points in meetings, status reporting, project schedules, milestone tracking, email triage, and plan analysis. I focused on practical AI use cases that would save time inside existing work habits instead of creating a separate dashboard users would ignore.

Solution

I designed a workflow that keeps AI useful and user-controlled

The final concept included priority inbox labels, response-needed detection, editable draft responses, meeting context summaries, email-to-Teams follow-up support, status report generation, project milestone tracking, and user-controlled review before sending.

Impact

Less Overload

Helped users identify which messages needed attention and which could wait.

Better Follow-Up

Improved visibility across emails, meetings, and project actions.

Clearer Updates

Helped turn scattered context into status-ready summaries.

Reflection

What I learned

This project strengthened my ability to define practical AI use cases around real work pain points. It also helped me think through trust, user control, and adoption barriers when designing AI tools for professional environments.
